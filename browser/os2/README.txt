================================================================================

Firefox を使う時は同梱の許諾契約に従ってください。

この Read Me ファイルにはシステム要件及び OS/2 版 Firefox のインストール手順に
ついての情報が含まれています。

Firefox についてもっと知りたい方は http://mozilla.jp/firefox/ を参照してください。
OS/2 版についての情報は http://www.mozilla-japan.org/ports/os2 を参照してください。
バグレポートやその他のフィードバック送るときは、まず Bugzilla 
(https://bugzilla.mozilla.org) の既知バグ情報、バグ報告ガイドラインやその他の
情報を確認してください。Bugzilla の使い方に関する手助けが必要な時は、IRC
クライアントで irc.mozilla.org の #mozillazine にアクセスしてもよいでしょう。
OS/2に固有の問題に関しては #warpzilla や news.mozilla.org の
mozilla.dev.ports.os2 ニュースグループで論議されています。


================================================================================

                             Firefox の入手方法

================================================================================

Firefox の公式マイルストーンビルドは以下のリリースページにて公開されます。

  http://mozilla.jp/firefox/releases/

OS/2 版は Mozilla.org のスタッフではない有志によって作成されており、この
リリースページより先に http://www.mozilla-japan.org/ports/os2 に公開されること
があります。リリースページからリンクされる Firefox のリリースノートを参照し、
Firefox の既知の問題やインストールに関する問題点を必ず確認するようにしてください。


================================================================================

                           OS/2におけるシステム要件

================================================================================

- このリリースを実行するためには
  ftp://ftp.netlabs.org/pub/gcc/libc-0.6.3-csd3.zip
  で公開されているCランタイムDLL (libc-0.6.3) が必要です。このパッケージを
  Firefox の実行ファイルと同じ場所もしくは LIBPATH 上のどこかに展開してください。

- 最小ハードウェア要件
  + Pentium 400 MHz クラスのプロセッサ。
  + 128 MiB のメモリと 128 MiB のスワップ領域。
  + インストールのための 40 MiB のハードディスク空き領域と、
    さらに、ディスクキャッシュに使用される記憶領域。
  + 256色より多い色数を表示できるグラフィックカードとドライバ。

- 充分なパフォーマンスを得るための推奨ハードウェア
  + 1.0 GHzクラスのプロセッサ
  + 512 MiB のメモリと 128 MiB のスワップ領域
    補足：Firefox は、利用できる物理メモリ量が多いほどパフォーマンスと安定度が
    向上します。

- ソフトウェア要件
  + ロングファイルネームが利用可能なファイルシステムへの導入。
    (たとえば HPFS や JFS。FAT は不可。)
  + Fixpack 15 以降を適用した OS/2 Warp 4。
    (Warp 3 でも動作する可能性はありますが、サポートされません。)
  + MPTS バージョン 5.3 以降。
  + TCP/IP バージョン 4.1 以降。
  + INETVER コマンドの結果:
        SOCKETS.SYS=5.3007, AFOS2.SYS=5.3001, AFINET.SYS=5.3006
    補足：この INETVER より古いバージョンの MPTS や TCP/IP を利用しないでください。
    Firefox は古いスタックでも起動し普通に動作するように見えますが、MPTS の古い
    バージョンでは Firefox が必要とするいくつかの機能が正しく実装されておらず、
    クラッシュやデータ損失を引き起こす可能性があります。

  + コンビニエンスパッケージ 2 もしくは eComStation 1.0 以降であれば、
    最初からこのソフトウェア要件を満たしています。

- 便利なオプションソフトウェア
  以下のソフトウェアは Firefox の追加機能を有効にします。ダウンロード場所は
  機能の詳細と共に後述します。
  + Remote Workplace Server (RWS) バージョン 0.8
  + Doodle's Screen Saver (DSSaver) バージョン 1.8 以降


================================================================================

                               インストール手順

================================================================================

どのプラットフォームにおいても必ず空の(新しい)ディレクトリーに展開してください。
導入済みの古い Firefox に上書きインストールすることは、しばしば問題を引き起こし
ます。

補足: この手順は Firefox をビルドする方法を説明するものではありません。
ビルド方法に関する情報を得るには、以下のページを参照してください。

  http://developer.mozilla.org/ja/docs/Build_Documentation


OS/2でのインストール手順
------------------------

   OS/2 版の Firefox にはインストールプログラムがありません。インストールする
   には .zip ファイルをダウンロードし、以下の手順に従ってください。

     1. Firefox をダウンロードしようとしているサイトの "Zip" リンクをクリックし、
     ZIP パッケージをご使用のコンピュータにダウンロードします。
     ファイル名は、通常 firefox-x.x.x.en-US.os2.zip の "x.x.x" の部分が
     Firefox のバージョンで置き換えられた形になっています。

     2. ファイルをダウンロードしたディレクトリーを開き、お好みのツールで
     ファイルを展開してください。

     3. この展開手順により、展開先に指定された場所に "firefox" ディレクトリーが
     作成されることに留意してください。たとえば
        unzip firefox-3.0.en-US.os2.zip -d c:\firefox-3.0
     とすると Firefox は c:\firefox-3.0\firefox の下に展開されます。

     4. 古い Firefox に上書き展開「しないで」ください。これを行うと様々な問題を
     引き起こすことが知られています。

     5. Firefox を開始するには、Firefox を展開した場所を開き、C ランタイム DLL
     がそのディレクトリーにコピーされている（もしくは LIBPATH 上にあるディレク
     トリーにインストールされている）ことを確認した上で、Firefox.exe オブジェクト
     をダブルクリックします。


複数のバージョンを同時に実行する方法
------------------------------------

数ある Mozilla プログラムファミリー (Mozilla, Firefox, Thunderbird, IBM Web
Browser など) が互換性のない同名別バージョンの DLL を使用しているため、
これらのプログラムを同時に実行するにはいくつかの追加手順が必要になります。

一つの回避方法は LIBPATHSTRICT 環境変数を利用することです。次の例のような
コマンドファイルを用意します。(この例では、Firefox は d:\internet\firefox に
インストールされているものと仮定します)

   set LIBPATHSTRICT=T
   rem 次の行は、別の Mozilla プログラムが LIBPATH に存在する場合に必要です。
   rem set BEGINLIBPATH=d:\internet\firefox
   rem 次の行は、2つのバージョンの Firefox を実行する場合にのみ必要です。
   rem set MOZ_NO_REMOTE=1
   d:
   cd d:\internet\firefox
   firefox.exe %1 %2 %3 %4 %5 %6 %7 %8 %9

同様に、Firefox を起動するためのプログラムオブジェクトを以下の設定で作成する
ことも出来ます。

   パスおよびファイル名: *
   パラメーター:         /c set LIBPATHSTRICT=T & .\firefox.exe "%*"
   作業ディレクトリー:   d:\internet\firefox

(システムの構成によっては、前述のコマンドファイルのように MOZ_NO_REMOTE や
BEGINLIBPATH を追加する必要があります。)

しかしながら、Hobbes ソフトウェアアーカイブに置いてある Rich Walsh の Run!
ユーティリティーを使うのが最も簡単な方法と言えるでしょう。

   http://hobbes.nmsu.edu/cgi-bin/h-search?key=Run!

詳しくは同梱のドキュメントを参照してください。


プロファイルをインストールディレクトリーから分離する方法
--------------------------------------------------------

MOZILLA_HOME 環境変数をお好みのディレクトリーに設定すれば、ユーザープロファイル
（ブックマークやカスタマイズ情報が含まれる）の場所をインストールディレクトリー
とは別の場所に配置する事が出来ます。
そのようにしておけば、ZIP パッケージで Firefox をアップデートする際にも
ユーザー設定を保持しておくことが容易になるでしょう。
環境変数は Config.sys で設定しても良いし、先の例のようにコマンドファイルや
プログラムオブジェクトで設定することも出来ます。たとえば

   set MOZILLA_HOME=f:\Data

とすると、Firefox のユーザープロファイルは f:\Data\Mozilla\Firefox に
作成されます。

Mozilla からの移行を行う場合、MOZILLA_HOME が既存の Mozilla プロファイルデータ
を正しく指し示している場合のみ、Firefox のインポート機能はそれを見つけることが
出来ます。


その他の重要な環境変数
----------------------

OS/2 版 Firefox の特別な動作を制御することが出来る環境変数が他にもいくつか
存在します。

- set NSPR_OS2_NO_HIRES_TIMER=1
  この設定を行うと Firefox は OS/2 の高分解能タイマー使用しません。
  高分解能タイマーを利用する他のアプリケーション（マルチメディア
  アプリケーション）の挙動がおかしいときに設定してください。

- set MOZ_NO_REMOTE=1
  2つの Firefox インスタンス（たとえばデバッグバージョンと最適化バージョン）
  を同時に実行する場合に使用します。

- set MOZ_NO_RWS=1
  Remote Workplace Server サポート（下記参照）を禁止する場合に使用します。

さらなる関連情報やその他のヒントに関してはこちらを参照してください：
   http://www.os2bbs.com/os2news/Warpzilla.html


ブラウザーの WPS オブジェクトサポート
--------------------------------------

Firefox は Rich Walsh の Remote Workplace Server (RWS) ライブラリを通じて
ワークプレース・シェル オブジェクトにアクセスすることが出来ます。
これにより、ダウンロードしたファイルを開く際に WPS の関連付けに基づいた
ヘルパーアプリケーションを選択することが可能です。さらに、WPSアイコンが
ダウンロードダイアログおよびディレクトリー一覧に表示されます。

この機能を有効にするには、Firefox が RWS の DLL を見つけられる必要があります。
そのためには DLL が LIBPATH 上または Firefox のディレクトリーに置かれるか、
もしくは WPS クラスとして登録されていなければなりません。eComStation 2 は
そうなっています。
もし RWS をまだ持っていないなら、次の場所からダウンロードしてください。
   http://hobbes.nmsu.edu/cgi-bin/h-search?key=rws08dll

システム上に RWS が見つかれば、Firefox は自動的にそれを使用します。
RWS サポートを禁止する必要がある場合は MOZ_NO_RWS 環境変数を作成し、
1に設定してください。


内部クリーンアップのための休止タイマー
--------------------------------------

システムに Doodle's Screen Saver (DSSaver) v1.8 以降がインストールされていると、
Firefox はシステムの無操作状態を検出することが出来ます。
無操作状態が一定時間継続すると Firefox は履歴などのクリーンアップ作業を実施し、
また、ライブブックマークの再読み込みは休止されます。

DSSaverは次の場所からダウンロードできます。
   ftp://ftp.netlabs.org/pub/dssaver/dssaver_v18.zip


OS/2 のデフォルトブラウザーを設定する
-------------------------------------

デフォルトの Web ブラウザーは WPS URL オブジェクトを使って設定出来ます。
すべての URL オブジェクトはプロパティーノートブックにブラウザー実行ファイル
を選択する「ブラウザー」タブがあります。この設定に対する変更は「省略時値に設定」
ボタンを押すと直ちに保存され、すべての URL オブジェクトに反映されます。

以下の2つの小さなアプリケーションは、この手続きをさらに簡単にし、さらに
メールやニュース、インターネットチャットサービスに対するデフォルトプログラムを
設定する機能も併せ持ちます。

- ConfigApps
     http://hobbes.nmsu.edu/cgi-bin/h-search?key=configapps&sort=date

- Internet Application Integration
     http://7cities.net/~mckinnis/os2/


OS/2バージョンにおける既知の問題
--------------------------------

プラットフォームに依存しない問題は、それぞれのマイルストーンリリースの
リリースノートに記載されています。

- Firefox は、64kB を超えるテキストをクリップボードにコピーしようとすると
  ビープ音を発します。
  これは、多くのアプリケーション（システムエディター、EPM、VIOウィンドウ
  アプリケーション等）がこのサイズのテキストをペーストできないことを
  ユーザーに警告するためのものです。

- Firefox は Type1 や TrueType 以外の OS/2 フォント（WarpSans 等）を
  利用出来ません。WarpSans フォントの代わりに
     http://users.socis.ca/~ataylo00/creative/fonts/
  もしくは
     http://hobbes.nmsu.edu/cgi-bin/h-search?key=wpsu_ttf
  から入手できる Workplace Sans フォントを利用することを推奨します。

その他の既知の問題は OS/2 Mozilla ページにある「現在 Open な Warpzilla バグ」
リンクから辿れます。
     http://www.mozilla-japan.org/ports/os2/
