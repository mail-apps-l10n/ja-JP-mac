ポップアップを制御する

このドキュメントは、ポップアップを制御するためのすべての&pref.plural;について説明します。

このセクションでは:
  • ポップアップとは何か？
  • ポップアップブロック&pref.pluralCaps;

ポップアップとは何か？

ポップアップウインドウ (すなわちポップアップ) は、あなたの許可なしに自動的に現われるウインドウです。 それらのサイズは様々ですが、通常、画面全体を覆い隠すことはしません。 いくつかのポップアップは &brandShortName; ウインドウの手前に開き、 また一方では ウインドウの後ろ (ポップアンダー) に現われます。

ポップアップおよびポップアンダーは、&pref.pluralCaps;の コンテンツ パネル から制御することができます。 ポップアップブロックはデフォルトで有効になっているので、ポップアップが現れるのを防ぐために &pref.pluralCaps;を気にする必要はありません。

ポップアップをブロックした場合、通知バーとともにステータスバーにアイコン を表示します。通知バー内の &pref.pluralCaps; ボタンか、ステータスバー内のアイコンをクリックすると、 以下の選択肢を含むメニューが表示されます:

  • このサイトによるポップアップを許可/禁止する
  • ポップアップブロック&pref.pluralCaps;... を変更
  • ポップアップをブロックするとき、このメッセージ(通知メッセージ) を表示しない
  • (ブロックしたポップアップを表示)

ポップアップをブロックすると、一部の Web サイトの動作を妨げるかもしれません: 銀行業務サイトを含む一部の Web サイトは、ポップアップを重要な機能のために使用します。 すべてのポップアップをブロックすると、その機能を無効にしてしまいます。 特定の Web サイトのポップアップを許可するために、 そのようなサイトを許可されたサイト一覧に追加してください。

ポップアップブロックは必ずしも動作するとは限りません: &brandShortName; は大抵のポップアップをブロックしますが、 一部の Web サイトは想定外の方法でポップアップを表示するかもしれません。

ポップアップブロック&pref.pluralCaps;

ポップアップブロック&pref.pluralCaps;は &pref.menuPath; の コンテンツパネル 内にあります。

ここでは次のことができます:

  • ポップアップウインドウをブロックする: ポップアップブロックをすべて無効にするには、 この&pref.singular;のチェックを外してください。
  • 許可サイト: ポップアップの表示を許可するサイトの一覧です。 ダイアログには以下の選択肢があります:
    • 許可: 入力した Web サイトを例外一覧に追加します。
    • サイトを削除: 例外一覧から Web サイトを削除します。
    • すべてのサイトを削除: 例外一覧の Web サイトをすべて削除します。

注意: ポップアップブロックは必ずしも動作するとは限りません。 また、一部の Web サイトの動作を妨げるかもしれません。ポップアップブロックに関しての詳しい情報は、 ポップアップとは何か？ をご覧ください。
